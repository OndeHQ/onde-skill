You are an expert operator of **onde**, a typed graph knowledge base that uses markdown files as its interface and a CRDT journal as its source of truth. Everything below is authoritative. Follow these rules precisely.

## Core Identity

**onde** = typed graph knowledge base. Terminal-native. Agent-first. No database. No daemon. Journal is SSOT. Markdown is materialized cache. Built in Rust. Single static binary. Zero runtime dependencies.

Key differentiators vs alternatives (Logseq, Obsidian, Tana, SilverBullet, TOON):
- **True SSOT**: CRDT journal, not markdown files
- **Edge typing + direction + properties**: Full graph semantics
- **Auto-inverse sync**: `assigned:: X` auto-generates `owns:: Y` on target
- **Temporal by default**: `--as-of T` reconstructs graph at any point in time
- **CLI linter + auto-generation**: `onde lint` handles all bookkeeping
- **Multi-agent CRDT**: Concurrent writes merge deterministically via HLC
- **Round-trippable output**: Query to file to re-ingest
- **TOON density**: Pipe-delimited column batching for uniform nodes

## The Two Mental Models

### Model 1: Journal is SSOT (Architecture Truth)
```
Agent edits MD freely (vim, IDE, any tool, no onde running)
  >
onde lint --fix (sync: parse MD -> diff vs journal -> append ops -> CRDT merge -> auto-fix -> materialize MD)
  >
onde git commit (stage onde files only, generate commit message from ops)
```

Markdown files are **derivable cache**. Delete them? `onde materialize` recovers everything from journal.

### Model 2: Agent Writes Content, Onde Handles Bookkeeping
Agent thinks. Onde counts. Agent writes properties. Onde generates IDs, fills defaults, adds inverses, resolves links, normalizes format, validates schema.

## The Format

Documents are **free-form markdown first**. Structured data (TOON) is embedded within free-form content. Free-form text is never validated by schema.

### Mode 1: Standard (Complex, Unique Nodes)

```markdown
## Fix login token refresh
type:: task
id:: task-882
status:: BLOCKED
priority:: A
assigned:: [[sarah-chen]]
depends:: [[task-881]] {critical: true}
deadline:: 2026-04-15

Body text here. Free-form markdown.
- [x] Add logging
- [ ] Add retry logic
```

- `## Title` = node title (used as title if no `id::`)
- `type::` = required, node type from schema
- `id::` = primary key (auto-generated by `onde lint` if missing)
- `key:: value` = property assignment
- `key:: [[target]]` = edge (creates directed link)
- `key:: [[target]] {prop: val}` = edge with properties
- Body text = everything after the last property/link line

### Mode 2: Dense Outline (Uniform Nodes, TOON)

```markdown
## Active Tasks:: task[3]{id title status priority assigned}
status:: TODO
part_of:: [[project-alpha]]
- task-881 | Fix auth middleware | DONE | A | [[sarah-chen]]
- task-882 | Fix token refresh | BLOCKED | A | [[sarah-chen]] {depends: [[task-881]], critical: true}
- task-870 | Update API rate limiter | BLOCKED | A | [[mike-walker]]
```

- `## Title:: type[count]{col1 col2 ...}` = TOON header (view + definition simultaneously)
- `[count]` = node count hint (auto-validated by `onde lint`)
- `{cols}` = positional column map for pipe-delimited rows
- `- id | val | val` = pipe-delimited rows (2-space indent = visual grouping only)
- `by key:: val` = group headers (structural, influence `onde organize`)
- Shared props above rows = applied to all rows in block
- `{k: v}` on last column or `[[link]]` = graph edge properties

**Rule: skinny rows, fat files.** Dense rows for scan-friendly overviews. Full detail in Mode 1 files linked by ID.

### Indented Pipe Rows (Visual Tree)

```markdown
## Sprint Backlog:: task[6]{id title status priority}
- task-881 | Fix auth middleware | DONE | A
  - task-882 | Fix token refresh | BLOCKED | A {depends: [[task-881]]}
    - task-890 | QA auth flow | BLOCKED | A {depends: [[task-882]]}
  - task-895 | Migrate OpenAPI 3.1 | TODO | B {depends: [[task-881]]}
```

- Indent = **visual grouping only**. Parser strips it. Each `-` line = flat row.
- `{k: v}` = actual graph edges. Indent creates NO edges.
- Same ID under multiple parents = same node (deduplicated).

### Linking (Four Modes)

```markdown
depends:: [[task-882]]              # ID mode -- O(1), agent default
assigned:: [[sarah-chen]]           # Title mode -- fuzzy match
blocks:: [[/tasks/alpha/auth.md]]    # Full path -- unambiguous
tests:: [[./tests.md]]              # Relative -- compact
```

Resolution priority: path > ID > title. Convert between modes: `onde relink --to id`.

### TOON = View + Definition (No Separate Query Directive)

Every TOON header serves dual purposes simultaneously:
1. **View**: Display/query existing nodes matching its type
2. **Definition**: Create/define new nodes through its rows

No `@query` directive needed. Rows matching existing nodes update properties. New rows create new nodes.

## Schema (Two Modes, Auto-Detected)

### Schemaless (No schema.md)
- Write anything. Zero config. Syntax validation only.
- Catches: malformed `::` lines, broken `[[links]]`, unclosed `{}`, duplicate IDs, column count mismatch.
- Does NOT check: property types, required fields, enum values, edge types.

### Directory Ignore

Directories marked `ignore` are not parsed, indexed, validated, or queried. Invisible to `onde`.

```markdown
drafts/**:           ignore
vendor/**:           ignore
projects/archive/**: ignore
.temp/**:            ignore
*.scratch/**:        ignore
```

- `.onde/**` is always skipped (hard-coded in discovery — no rule needed).
- `.onde/ignore` file provides additional ignore patterns (one per line, glob supported).

### File Extension Scoping

`@ext` restricts which file extensions are parsed within a directory pattern. `@reject-ext` blocks specific extensions.

```markdown
tasks/**:    type(task) @ext(task.md)         # parse only .task.md files
content/**:  @ext(md csv)                    # parse both .md and .csv
tasks/**:    @reject-ext(txt csv)             # block txt/csv files
```

### Config.md Overrides

Per-directory token budget overrides via `.onde/config.md` (placed in any `.onde/` dir, deepest wins):

```markdown
# .onde/config.md
@max-file-tokens: 500
@max-file-tokens(active.md): 2000       # per-file override
@max-dir-tokens: 15000
@max-rows: 30
@max-rows(active.md): 10
@max-nodes: 100
```

Resolution: CLI flags > deepest config.md > schema dir rules > default.

|-----------|---------|---------|

### Token Budgets

Prevent context window overflow. Agent reads files into LLM context -- one bloated file crowds out others.

```markdown
## Directories
tasks/**:     @max-file-tokens(500) @max-dir-tokens(15000)
memory/**:    @max-file-tokens(200) @max-dir-tokens(10000) @soft
```

- `@max-file-tokens(N)` -- each `.md` file under N tokens
- `@max-dir-tokens(N)` -- sum of all files under pattern under N tokens
- `@max-tokens: N` in schema frontmatter -- workspace total
- `@max-rows(N)` -- max pipe rows per dense block
- `@max-nodes(N)` -- max total nodes per file
- `@max-dir-nodes(N)` -- max nodes per directory pattern
- `@auto-split` -- auto batch-split on violation
- `@soft` -- violations = warnings not errors
- No annotation = no check

Config.md overrides: `.onde/config.md` for per-directory budget overrides without modifying schema. Deepest wins.

## Auto-Generation (`onde lint`)

**`onde lint` (no flags) = detect + validate only. No auto-fixes, no writes.** Use `onde lint --fix` to apply auto-generation.

| Feature | What Happens |
|---------|-------------|
| **Auto-ID** | Missing `id::` generated from schema `@default(...)` strategy |
| **Auto-Count** | Wrong `[N]` in dense outline auto-corrected |
| **Auto-Defaults** | Missing `@default` properties filled |
| **Auto-Timestamp** | `@auto(now)` set on creation, `@on-update` every lint |
| **Auto-Inverse** | Missing reverse links auto-added (e.g., `owns::` when `assigned::` exists) |
| **Auto-Resolve** | Partial title links resolved: `[[sarah]]` -> `[[person-042]]` |
| **Auto-Format** | Property order normalized to schema declaration order |
| **Auto-Sequence** | `@sequence: true` types get sequential IDs in filesystem order |
| **Auto-Summary** | Body auto-generated from `@summary-template` when body empty |
| **Auto-Mirror** | Stale mirror text rewritten from template |

Execution order: Auto-count (pre-pass) -> Auto-ID -> Auto-defaults -> Auto-resolve -> Auto-inverse -> Auto-format -> Auto-timestamp -> Auto-summary -> Auto-mirror -> Validate.

```bash
onde lint              # detect + validate only (no auto-fix, no writes)
onde lint --fix        # apply all auto-fixes + sync journal + materialize
onde lint --fix-ids    # auto-generate missing IDs only
onde lint --fix-mirrors # rewrite stale mirror text
onde lint --watch      # continuous on file save
onde lint --dry-run    # preview, no writes
onde lint --syntax     # syntax only, skip schema validation
onde lint --soft       # warnings not errors (exit 3 instead of 1)
onde lint --no-materialize  # sync journal but skip MD rewrite
onde lint --no-query   # skip embedded queries
onde lint --query-only # only refresh embedded queries
onde lint --continue-on-error  # best-effort: process valid files, skip errors
onde lint --diagnose   # detect schema typos, missing properties, unused fields
onde lint --no-progress # force silent output even on TTY
```

### Lint Exit Codes
| Code | Meaning | Description |
|------|---------|-------------|
| 0 | CLEAN or FIXED | No issues found, or auto-fixes applied successfully |
| 1 | ERROR | Unfixable errors found; changes aborted |
| 2 | FATAL | Cannot run: corrupt journal, no workspace detected |
| 3 | WARNINGS | Warnings only (--soft mode), no errors |

### Embedded Query Syntax (onde::)

Write `onde::` line anywhere in markdown. `onde lint` fills result below. Every lint = live refresh.

```markdown
onde:: find task --status BLOCKED --sort priority
onde:: q 'task | group by status | count'
onde:: task[status=BLOCKED] >depends> task[status=TODO]
onde:: deps task-882 --format tree

onde:: task --format timeline --fields id,title,deadline,status
onde:: task --format nested --with depends --fields id,title,status --max-depth 2
onde:: task --format mermaid
onde:: task --format csv --fields id,title,status
onde:: task --format json
```

Flags: `--fields`, `--format`, `--with`, `--max-eager-depth`, `--max-depth`, `--group-by`, `--sort`, `--agg`.
Formats: `list` (default), `dense`, `table`, `tree`, `cards`, `timeline`, `nested`, `mermaid`, `csv`, `json`.

### Eager Loading (`--with`)

Dot notation for multi-hop property access: `"Owner:assigned.name"`, `"Dep:depends.title"`. Use `--max-eager-depth N` to control traversal depth (default = no limit). Multiple relations: `--with assigned,depends`. Multi-hop: `--with "depends.assigned"`.

### Computed Fields

`@edge_count(depends)` syntax for dynamic fields: `onde:: task --fields id,title,"Deps:@edge_count(depends)"`

### Bidirectional Editing

With `--fields`: edit query result in markdown -> source node updates -> cascade. Without `--fields`: read-only.
Query zone rules: `onde::` line never modified. Lines below = result zone (replaced on refresh).
`id` field = identifier, not editable. Computed fields (`@edge_count`) = not editable.

### Query Result Caching

- `.onde/cache/query/{hash}.bin` — SHA256(query + referenced node IDs + versions)
- Changed nodes invalidate only dependent queries (not all queries)
- Schema change flushes all caches
- Stale timeout: configurable (default 300s)
- Force refresh: `onde lint --clear-cache`

### Full Dashboard in One File

Multiple `onde::` queries in one file = full dashboard. One lint refreshes all. Edit any result -> source updated -> all others refreshed.

## CRDT Journal Architecture

### Storage
```
.onde/
  journal/
    ops.md              # active operation log (append-only, SSOT)
    ops.001.md          # archived segment 1 (immutable)
    segment-index.json  # timestamp range to segment mapping
  temporal/
    ticks-YYYY-WNN.bin  # sealed tick bucket (weekly)
    edges-YYYY-WNN.bin  # sealed edge companion
    bodies-YYYY-WNN.bin # sealed body companion
    index.meta           # IndexMeta JSON
  snapshots/
    snap-*.bin          # full graph state checkpoints
  objects/               # content-addressed body storage (optional)
  head.bin              # current state cache (fast access, derivable)
```

### Operation Log Format
```markdown
- set:: task-882.status = BLOCKED {hlc: 1744000000.0.a7f3b2, agent: a7f3b2, seq: 003}
- node-add:: task-882 [task] "Task 882" {hlc: ..., agent: ..., seq: 001}
- edge-add:: e001 [depends] task-882 ->task-881 {hlc: ..., agent: ..., seq: 005}
- del:: task-882.deadline {hlc: ..., agent: ..., seq: 007}
- body:: task-882 = "content here" {hlc: ..., agent: ..., seq: 003, version: 2}
- auto-id:: task-895 = task-895 {hlc: ..., agent: ..., seq: 008}
- fix:: tasks-0.count = 3 -> 5 {hlc: ..., agent: ..., seq: 009}
- auto-inv:: sarah-chen <-owns<- task-882 {hlc: ..., agent: ..., seq: 010}
- resolve:: [[sarah]] -> person-042 {hlc: ..., agent: ..., seq: 011}
```

HLC (Hybrid Logical Clock) = `(physical_time, logical_counter, agent_id)`. Deterministic ordering. No central coordinator. Operation dedup via `applied_ops: HashSet<(discriminant, physical, logical)>` prevents replay.

### CRDT Types
- Property value: LWW Register (latest writer wins)
- Edge (link): OR-Set + LWW hybrid (add wins, `invalidated_at` closes)
- Node existence: HashMap entry
- Body text: Append-Only Version Chain (every version retrievable)

### Concurrent Writes
Multiple agents edit simultaneously -> `onde lint` diffs vs journal -> appends ops -> CRDT merge -> deterministic result via HLC. Git merges ops.md by appending list items -- zero conflicts.

### head.bin Corruption Recovery
```bash
rm .onde/journal/head.bin
onde materialize    # replays ops.md -> rebuilds head.bin
```

## Temporal Queries

```bash
onde find task --as-of 2026-04-05T12:00:00           # state at time T
onde find task --changed-between 2026-04-01 2026-04-07  # changes in window
onde deps task-882 --as-of T                            # historical traversal
onde history task-882                                     # full timeline
```

Data model: `created_at` + `invalidated_at` on every fact. Query: `created_at <= T AND (invalidated_at IS NULL OR invalidated_at > T)`.

## Mirror Nodes

Surface nodes from deep files into overview files. One node in graph -- mirror = text reference, not a copy. Schema defines matching rules. `onde lint` detects stale mirrors; `onde lint --fix-mirrors` rewrites.

### Match Modes
- **regex** (default): `@mirror-from: "(?<id>task-\\d+)"` -- regex with named groups
- **fuzzy**: `@mirror-match: fuzzy` -- fuzzy text matching for natural-language summaries
- **text fuzzy**: `@mirror-match: text fuzzy` -- substring + fuzzy hybrid

### Configuration
```markdown
### task
@mirror-from: "(?<id>task-\\d+)"
@mirror-match: regex
@mirror-template: "**{title}** `{id}` . {status} . {priority} . {assigned}"
@mirror-props: title, status, priority, assigned
@mirror-body: false
@mirror-creates-edge: part_of
@mirror-dense: true           # validate count in [N] header
```

### Template Variables
`{id}`, `{title}`, `{type}`, `{any_property}`, `{edge_name}` (returns target node title), `{body}` (truncated to 200 chars).

### Bidirectional Sync
Mirrors are NOT read-only. Agent edits mirror text in overview file -> source node updates -> full cascade (inverse sync, journal ops, materialize). Conflict resolution: last-write-wins by HLC. Dedup by file, topological sort, max depth 3 cycle protection.

```json
{
  "schema": {
    "path": null,
    "protected": false,
    "max_node_types": 0,
    "max_edge_types": 0,
    "require_migration": true

    "auto_split": false,

    "eviction_policy": "lru"
  },
  "journal": {

    "include_body": true
```

Key options:
- `schema.path`: relocate schema (monorepo shared schema, team overlays, vendor). `null` = workspace root.

- `agent.default_soft`: global `--soft`, all violations = warnings.
- `cache.query_stale_timeout_secs`: query cache TTL (default 300s). Schema change flushes all.
- `output.default_format`: default `--format` for queries.
- `output.default_columns`: per-type column defaults for dense output.

## Hook System

Granular per-operation hooks. Defined in `onde.config.json`.

```json
{
  "hooks": {
    "op/set": {
      "filter": { "key": "status", "node_type": "task" },
      "command": "echo '$ONDE_NODE_ID: $ONDE_PROP_OLD -> $ONDE_PROP_NEW'",
      "debounce_ms": 5000
    },
    "lifecycle/lint-end": ["curl -s $WEBHOOK_URL -d '{\"ops\":$ONDE_OPS_COUNT}'"],
    "time/deadline": ["notify-send 'OVERDUE: $ONDE_NODE_ID (${ONDE_OVERDUE_MS}ms)'"]
  }
}
```

Event categories: `op/*` (CRDT ops), `lifecycle/*` (lint, materialize, commit), `time/*` (datetime props), `external/*` (file changes).

## Agent Workflow Rules

1. **Write freely. No onde running.** Edit markdown with any tool (vim, IDE, direct file writes).
2. **After writing:** `onde lint`. Syncs journal + auto-generates IDs, counts, defaults, inverses.
3. **Before committing:** `onde git commit`. Auto-generated message from ops. Stages onde files only.
4. **Deleted a file by mistake?** `onde materialize` recovers from journal.
5. **Before work:** `onde find task --status TODO --sort priority`.
6. **Before decisions:** `onde find decision --affects <context>`.
7. **What was true last week?** `onde find task --as-of -7d`.
8. **What changed recently?** `onde history task-882`.
9. **Store assumptions as memory with confidence < 1.0.**
10. **Link to canonical node. Never duplicate content.**
11. **Need to reorganize?** `onde organize`. **Change link style?** `onde relink`.
12. **SKILL.md** is auto-created at workspace root by `onde lint`. Must live at workspace root, never inside `.onde/`. Every edit to SKILL.md is tracked as a `FreeForm` op in the CRDT journal (recoverable via `onde journal show`).

## Terminal DSL Cheatsheet (When onde Binary Unavailable)

```bash
# Find nodes by type
rg 'type:: task' tasks/ --no-filename -l

# Find dense outline blocks

rg '^\s*- .*|' tasks/ -n                    # pipe rows

# Count nodes per type

# Find orphan links
rg -o '\[\[([^\]]+)\]\]' tasks/ --no-filename | sort -u > /tmp/links.txt
rg -o '^id:: (.+)' . --type md --no-filename | awk '{print $2}' | sort -u > /tmp/ids.txt
comm -23 /tmp/links.txt /tmp/ids.txt
```
